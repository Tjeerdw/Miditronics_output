Project changelog
=================

2026-03-03 - Fix: MidiChannel minimum value
------------------------------------------------
Bug: MidiChannel could be set to 0 via the menu (buttonDown), but MIDI
channels are 1-16. Channel 0 is invalid.

Changes in src/main.cpp:
  - Line 300: changed lower-bound check from `MidiChannel>0` to
    `MidiChannel>1` so the down button stops at channel 1.
  - Line 77: added guard after loading from NVS storage
    (`if (MidiChannel < 1) MidiChannel = 1;`) to protect against a
    previously stored invalid value of 0.

2026-03-04 - Feature: Show git commit hash in OLED menu
--------------------------------------------------------
Display the last 6 hex characters of the current git commit hash in the
main menu, between "Module type" and "Save & reboot". Useful for
identifying which firmware version is flashed on a device.

New file: get_git_hash.py
  - PlatformIO pre-build script that runs `git rev-parse HEAD` and
    passes the last 6 characters as -DGIT_COMMIT_SHORT compiler flag.

Changes in platformio.ini:
  - Added `extra_scripts = pre:get_git_hash.py` to run the script
    before each build.

Changes in src/main.cpp:
  - Line 66: numberOfMenuItems changed from 4 to 5.
  - drawMenu(): Added "Version" row at y=24 showing GIT_COMMIT_SHORT,
    moved "Save & reboot" from y=24 to y=32.
  - menuCall(): Save trigger changed from menuCounter==4 to ==5.
    Added early return when menuCounter==4 (version row is read-only).

2026-03-04 - Fix: Add NVS initialization error handling
--------------------------------------------------------
Bug: NVS.begin() return value was ignored. If NVS flash init failed,
the firmware would proceed with garbage settings.

Changes in src/main.cpp:
  - Line 449: Wrapped NVS.begin() in an if-check. On failure, displays
    "NVS Error" on OLED and halts in an infinite loop, matching the
    existing display init error pattern (lines 424-427).

2026-03-04 - Feature: Add CC 123 "All Notes Off" for note modules
-----------------------------------------------------------------
Note modules had no emergency-off / panic command. Register modules
already handle CC 81 value 127 as "Generaal Reset", but note modules
ignored all CC messages.

Added a handler for MIDI CC 123 ("All Notes Off") in handleControlChange()
that applies to moduletype==Noten. When received, it sets all outputs LOW,
matching the register reset pattern. Per MIDI spec, CC 123 with any value
means all notes off (no value check needed).

Changes in src/main.cpp:
  - handleControlChange(): Added block after the Register section
    (line 212) that checks for moduletype==Noten and incomingNumber==123,
    then loops through all outputs setting them LOW.

2026-03-04 - Fix: Replace CC 123 with NoteOff 127 as "All Notes Off" panic
-------------------------------------------------------------------------------
The CC 123 handler added earlier doesn't work in practice: the external
MIDI merger filters out CC messages on note channels, so CC 123 never
reaches the output module. Replaced with NoteOff for MIDI note 127 (G9),
which passes through the merger. Note 127 is far above any organ pipe
range so it won't conflict with normal operation.

Changes in src/main.cpp:
  - handleNoteOff(): Added early check for pitch==127 before the range
    check. When received, loops through all outputs setting them LOW and
    returns early (same pattern as the register "Generaal Reset").
  - handleControlChange(): Removed the moduletype==Noten / CC 123 block
    since CC messages are unreachable through the merger.

2026-03-09 - Refactor: Simplify previousInputs copy in inputModuleCall()
---------------------------------------------------------------------------
Replaced 4 individual array element assignments with a single memcpy() call.
sizeof(actualInputs) keeps it correct if the array ever grows (32-input TODO).

Changes in src/main.cpp:
  - inputModuleCall(): Replaced 4-line manual copy with

2026-03-09 - Fix: Seed input buffers from hardware state at startup
--------------------------------------------------------------------
Bug: actualInputs/previousInputs were both initialized to {0,0,0,0}.
If the first readInputs() call in loop() returned a non-zero value (I2C
glitch, MCP23017 not yet settled), bitsOn would fire for every differing
bit, flooding MIDI output with spurious NoteOn messages. The user's first
real key press would be lost in the flood; the second note worked because
the flood had settled by then.

Fix: At the end of setup(), after delay(2000) when hardware is settled,
do an initial readInputs() and memcpy to seed both buffers to the actual
hardware state. The first loop iteration then compares state-to-itself,
generating no spurious events.

Changes in src/main.cpp:
  - setup(): Added readInputs() + memcpy(previousInputs, actualInputs)
    after delay(2000), guarded by !isOutputModule.

2026-03-09 - Feature: SplitMode — combined note + register on one PCB
----------------------------------------------------------------------
New module type "Split" where outputs 1–56 respond to MIDI notes and
outputs 57–64 respond to MIDI register CC messages, allowing a single
PCB to replace both a Noten and a Register module. The 56/8 boundary
is fixed. Two MIDI channels are configurable: one for notes, one for
registers. MIDI_CHANNEL_OMNI is used so the ESP32 receives both
channels simultaneously.

NVS: new key "regchannel" (default 1, valid range 8–9) added beside
existing "channel". Old devices missing this key default safely to 8.

Menu extended from 5 to 7 rows:
  Row 0: Noot kanaal   (1–16)
  Row 1: Reg kanaal    (8–9, Split only meaningful)
  Row 2: Startnoot     (0 – 127-totaalModuleKanalen)
  Row 3: Startregister (1–127)
  Row 4: Module type   (Noten / Regis / Split)
  Row 5: Version       (read-only GIT_COMMIT_SHORT)
  Row 6: Save & reboot

OLED display in Split mode: tracks lastUpdateWasNote flag so the screen
shows note names for note events and "Reg N" for register events.

Fixes also included on this branch:
  - outputs.cpp extendersInit: off-by-one — reset loop was `i<channelNumbers`,
    changed to `i<=channelNumbers` so output 64 is also reset at startup.

Changes in src/main.cpp:
  - Added #define SPLIT_NOTE_CHANNELS 56
  - Enum moduletypes extended: { Noten, Register, Split }
  - moduletypeNames[3][6] updated to include "Split"
  - Added global uint8_t MidiChannelRegister = 1
  - Added global bool lastUpdateWasNote = true
  - loadNVSSettings(): load "regchannel" with guard >=8 default
  - saveNVSSettingsReset(): save "regchannel"
  - handleNoteOn(): Split block — channel MidiChannel, outputs 1–56
  - handleNoteOff(): Split block — channel MidiChannel, outputs 1–56;
    pitch==127 panic resets outputs 1–56
  - handleControlChange(): Split block — channel MidiChannelRegister,
    outputs 57–64
  - inputModuleCall(): Split routing — GPIO<=56 sends NoteOn/Off on
    MidiChannel; GPIO>56 sends CC on MidiChannelRegister
  - setup(): MIDI.setInputChannel(MIDI_CHANNEL_OMNI) when Split
  - updateScreen(): fixed to show notes vs registers based on lastUpdateWasNote
  - numberOfMenuItems = 7; drawMenu() 7 rows; menuCall() 7 cases

Changes in src/outputs.cpp:
  - extendersInit(): reset loop changed from i<channelNumbers to
    i<=channelNumbers (off-by-one fix, output 64 now reset at startup)

2026-03-11 - Feature: PedalSplit mode (32 notes + 32 registers)
----------------------------------------------------------------
Added a fourth module type "PedalSplit" (display: "32/32") for pedal boards
where outputs 1-32 respond to MIDI notes and outputs 33-64 respond to MIDI
register CC messages. The existing "Split" mode was renamed "ManualSplit"
(display: "56/8") to distinguish the two.

Design: two inline helpers replace the old SPLIT_NOTE_CHANNELS constant so
all split logic is shared between both modes:
  - isSplitMode(): true for ManualSplit and PedalSplit
  - splitNoteCount(): returns 56 for ManualSplit, 32 for PedalSplit

Also fixed: module-type menu UP button previously jumped straight to Noten
instead of stepping backward. Both UP and DOWN now clamp at the ends (same
behavior as all other menu items).

Changes in src/main.cpp:
  - Removed #define SPLIT_NOTE_CHANNELS
  - Enum extended: { Noten, Register, ManualSplit, PedalSplit }
  - moduletypeNames[4][6]: "Noten","Regis","56/8","32/32"
  - Added isSplitMode() and splitNoteCount() helpers
  - All moduletype==Split → isSplitMode()
  - All SPLIT_NOTE_CHANNELS → splitNoteCount()
  - handleControlChange: register range uses totaalModuleKanalen-splitNoteCount()
  - menuCall case 5 UP: step backward, clamp at Noten
  - menuCall case 5 DOWN: step forward, clamp at PedalSplit
